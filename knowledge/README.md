# Knowledge base

Your long-term learning notebook for this training repo.

Everything goes through **`/teacher`** — teaching and knowledge tracking in one place.

## How it works

| Folder / file | Purpose |
|---|---|
| `PROGRESS.md` | Living dashboard: what you can do, what you are learning, next goals |
| `glossary.md` | Hard words → simple meanings |
| `concepts/` | Things you understand (one file per idea) |
| `open-questions/` | Things you are unsure about (until answered) |
| `mistakes/` | Errors you hit and what they taught you |
| `terminal/` | Linux commands you actually use while learning |
| `sessions/` | Short logs after a teaching session (optional) |

## Workflow (only `/teacher`)

1. `/teacher …` — ask a question → get **one next idea** (terminal-first).
2. Knowledge updates automatically when you are unsure, get it, hit a useful error, or learn a word/command.
3. Quick phrases (still `/teacher`):
   - `/teacher progress` — where am I?
   - `/teacher unsure: …` — save an open question
   - `/teacher I get it: …` — save a concept
   - `/teacher mistake: …` — save an error lesson
   - `/teacher save` — session crumb
4. `PROGRESS.md` stays the map of where you are.

## Naming rules

- Files: `kebab-case.md` (example: `if-else.md`, `javac-and-java.md`)
- One main idea per file
- Keep entries short — future-you should skim in 30 seconds
- Prefer your own words (kid-simple). Do not paste full homework solutions here.

## Related old notes

Free-form notes still live in `Lernnotizen.txt` at the repo root.  
Over time, move solid ideas from there into `concepts/` or `glossary.md`.

```text
concepts/

```

Start flat; add subfolders only when a topic has 5+ files.
